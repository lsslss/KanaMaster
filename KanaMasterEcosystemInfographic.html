Kana Master Ecosystem

A next-generation Japanese learning platform powered by AI generation, high-fidelity neural voice synthesis, and offline-first mobile architecture.

Core System Architecture

The ecosystem is built upon five highly integrated components, ensuring seamless content creation, voice generation, and edge delivery.

Mobile Learning Client

React 18 SPA (index.html v4.2) acting as the terminal interface. Features offline High-Fidelity audio via IndexedDB and intelligent fallback strategies.

Voice Manager Hub

(Manager.html v4.6) Automates gap analysis and batch generation of neural audio. Supports VOICEVOX, Google Proxy, and Gemini APIs with CORS bypass.

GAS Backend Service

(code.gs v4.2) The crucial bridge connecting Google Sheets to Google Drive. Provides endpoints for JSON syncing and audio Base64 retrieval.

AI & Data Spec

Lesson Spec v3.0 defines a strict JSON/CSV schema. The GEM Prompt v3.2 instructs LLMs to generate perfectly formatted lessons, ensuring one-to-one kanji/reading mapping.

TTS Engine Performance Analysis

Evaluating the three supported Text-to-Speech engines integrated within the Manager Hub across key operational metrics.

VOICEVOX

Local processing. Unmatched realism and zero rate limits, but requires local computing power.

Google Proxy

Extremely fast and highly reliable. Ideal for massive batch gap-filling where tone is secondary.

Gemini TTS

Studio-grade quality via API, restricted by strict 10 RPM limits on free tiers.

The Hybrid Fallback Strategy

Kana Master guarantees an uninterrupted learning experience through a tiered, fault-tolerant audio delivery workflow.

Priority 1: Local Cache

IndexedDB checks for downloaded Voice Packs. Zero latency, offline capable.

Priority 2: Cloud Stream

Fetches Base64 audio via GAS proxy from Google Drive cache. Requires connection.

Priority 3: System TTS

Ultimate fallback using the browser's Web Speech API. Ensures practice never stops.
